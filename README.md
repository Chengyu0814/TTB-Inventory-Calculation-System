# 飛買家銷售資料彙整工具

一個全端網頁應用程式，用於自動彙整電商平台的 Excel 銷售明細，依 SKU 加總銷售量、銷售額，並整合庫存、成本等資料，匯出彙整後的報表。

網站：https://zeabur-test1.zeabur.app/

## 功能介紹

- **上傳多種 Excel 來源** — 支援銷售明細、在途庫存、機上量、期末存量、本月進貨、商品成本，各資料可單獨或混合上傳
- **自動彙整資料** — 依 SKU 分組，加總各品項的銷售量與銷售額
- **自動偵測月份** — 從檔名（如 `20260301-20260327`）自動判斷月份，輸出對應欄位名稱（如「三月銷售量」）
- **幣別匯率換算** — 上傳商品成本時，自動偵測非台幣幣別並提示輸入匯率，統一轉換為 TWD
- **即時下載報表** — 處理完成後自動下載彙整結果 Excel 檔（單一工作表「報表」）

## 使用方式

### 輸入檔案格式

以下各類檔案均為 `.xlsx` / `.xls` 格式，皆可單獨或任意組合上傳。

#### 銷售明細（可多份，每份對應一個月份）

| 欄位 | 說明 |
|------|------|
| `SKU no` | 商品 SKU 編號 |
| `SKU title` | 商品名稱 |
| `Volume` | 銷售量 |
| `Amount` | 銷售額 |

> 若 Excel 內有多個工作表，程式會自動尋找名為 `details` 的工作表。

#### 採購未交量（在途庫存）

| 欄位 | 說明 |
|------|------|
| `品    號` | 商品品號（row 4 起始） |
| `品   名` | 商品名稱 |
| `未交數量` | 在途庫存數量 |
| `交貨庫` | 篩選條件（僅保留 `華膳-IT`） |

#### 機上量（需同時上傳一般航線與串飛航線）

| 欄位 | 說明 |
|------|------|
| `SKU No.` | 商品 SKU 編號（row 2 起始） |
| `數量` | 裝載數量 |
| `DESCRIPTION` | 商品描述 |

> 一般航線數量 × 41；串飛航線數量 × 10，兩者加總為機上量。

#### 期末存量

| 欄位 | 說明 |
|------|------|
| `料號` | 商品料號（row 6 起始） |
| `品名` | 商品名稱 |
| `期末存量` | 期末庫存數量 |

#### 進貨明細（本月進貨）

| 欄位 | 說明 |
|------|------|
| `品號` | 商品品號（row 4 起始） |
| `品名` | 商品名稱 |
| `驗收數量` | 本月進貨數量 |
| `庫別` | 篩選條件（僅保留 `華膳-IT`） |

#### 品號價格資料（商品成本）

| 欄位 | 說明 |
|------|------|
| `品號` | 商品品號（row 3 起始） |
| `品名` | 商品名稱 |
| `幣別名稱` | 採購幣別（非台幣者需輸入匯率） |
| `採購單價` | 商品採購單價 |
| `核價日` | 以最新一筆核價紀錄為準 |

### 輸出檔案格式

匯出單一 Excel 檔，工作表名稱為「報表」，欄位依序如下：

| 欄位 | 說明 | 來源 |
|------|------|------|
| `SKU No.` | 正規化後的 SKU 編號 | 所有來源 |
| `品名` | 商品名稱 | 優先順序：銷售明細 > 在途庫存 > 機上量 > 期末存量 > 本月進貨 > 商品成本 |
| `TWD成本` | 換算為台幣的採購單價 | 商品成本 |
| `機上量` | 一般航線 + 串飛航線加總裝載數 | 機上量 |
| `{月份}銷售量` | 當月銷售量合計（可多月） | 銷售明細 |
| `{月份}銷售額` | 當月銷售額合計（可多月） | 銷售明細 |
| `期末存量` | 期末庫存數量 | 期末存量 |
| `本月進貨` | 本月進貨數量 | 進貨明細 |
| `在途庫存` | 採購未交數量 | 採購未交量 |

- 有銷售明細時，檔名為 `TTW sales summary {起月}-{迄月}.xlsx`
- 無銷售明細時，檔名為 `TTW 庫存表.xlsx`

## API

| 方法 | 路徑 | 說明 |
|------|------|------|
| `GET` | `/` | 健康檢查 |
| `POST` | `/scan-cost-currencies` | 掃描商品成本檔案中的幣別，回傳非台幣幣別清單 |
| `POST` | `/process-excel` | 上傳各類 Excel 並回傳彙整結果 |

## 本地開發

```bash
# 安裝後端相依套件
cd backend
pip install -r requirements.txt

# 啟動後端（port 8080）
uvicorn main:app --host 0.0.0.0 --port 8080 --reload
```

前端為靜態檔案，直接用瀏覽器開啟 `frontend/index.html`，或以任意靜態伺服器提供服務即可。
